# 自动化内容更新与 SEO 策略构想

本文档记录了基于 Next.js 静态导出 (`output: 'export'`) 架构下，实现每日自动更新文章以提升 SEO 效果的两种构想及最佳实践策略。

## 核心前提
本项目采用 Next.js 纯静态导出，没有后端 Node.js 服务。因此，**任何内容的更新都必须配合“自动构建（Daily Rebuild）”流程**（例如通过 GitHub Actions 或 VPS 定时任务执行 `npm run build`），才能将最新数据渲染成静态 HTML 文件。

---

## 策略构想对比

### 方案一：“预生成 + 定时释放”（细水长流策略）
适用于：常青内容（Evergreen Content）、基础科普、使用教程、疑问解答等不受时效性影响的深度文章。

*   **执行方式**：
    1. 预先通过 AI 或其他方式生成大量（如 180 篇）文章，格式为 JSON 或 Markdown。
    2. 为每篇文章设定 `publishDate`（发布日期），存入项目库。
    3. 在 Next.js 路由生成逻辑中，过滤并**仅渲染 `publishDate <= 当前日期` 的文章**。
    4. 每日通过定时任务触发 `npm run build`。由于日期推进，符合条件的文章数量逐渐增加。
*   **优缺点**：
    *   **优点**：极其稳定、省心。一次性排雷，发布过程不受外部网络或 API 抖动影响；适合建立网站权重的基础盘。
    *   **缺点**：缺乏时效性，无法跟进实时热点行情。

### 方案二：“动态抓取 + AI改写”（自动化流水线策略）
适用于：短时效资讯、币种行情、平台活动快讯。

*   **执行方式**：
    1. 编写独立爬虫脚本（Python/Node.js），每日定时抓取指定资讯源（不限于币圈媒体、快讯）。
    2. 将抓取内容输入 AI API（如 OpenAI/Gemini/Deepseek）进行“洗稿”或总结，生成原创口吻的简明短讯（约 100 字）。
    3. 脚本将改写后的内容追加写入项目的本地数据文件（如 `alerts.json` 或 `timeline.json`）。
    4. 脚本触发项目的 `npm run build` 生成最新静态页面。
*   **优缺点**：
    *   **优点**：内容时刻保持最新，蜘蛛抓取频率高，有利于触发搜索引擎的新鲜度（Freshness）奖励。
    *   **缺点**：维护成本高，极度依赖源站结构稳定性和 AI API 的稳定性，容易因抓取失败或 AI 幻觉导致构建出错。

## 最佳实践推荐：融合策略与“超级权威页（Pillar Page）”模型

为了最大化 SEO 收益，同时控制开发维护成本，建议采用 **“80% 预生成深度长文 + 20% 动态短快讯”** 的融合策略。

对于难度较高、最容易吸引爬虫复访的“短快讯”（方案二），**强烈建议遵循以下 SEO 原则进行页面组织，切勿为百字短讯建立独立 URL。**

### “时间轴/行情直播间”聚合模式
*   **避免死穴**：如果每天为 100 字快讯新建如 `/news/btc-drop` 页面，会被搜索引擎判定为“薄弱内容（Thin Content）”而遭到屏蔽或降权。
*   **打造核心页**：建立统一的主题聚合页，例如 `/crypto-market-updates`（加密市场动态）。
*   **数据追加（Append）机制**：
    *   每日脚本生成的 100 字短评，直接 `unshift`（插入最前）到该聚合页的 JSON 数据源中。
    *   页面内通过“信息流/时间轴”的样式展现最新的 10-50 条资讯。
*   **SEO 收益放大**：
    1.  **页面厚度随时间增长**：今日 100 字，两月后就是几千字的高度相关长文，极大力度增强该单页（对于某一核心关键词，如“BTC价格分析”）的权威性权重。
    2.  **触发 Freshness Score**：旧 URL 持久、高频的更新，是明确地向爬虫传达“此页面受到积极维护”的信号，有利于排名持续提升。
    3.  **内链引擎**：最新的 3-5 条短讯可提取为 Widget 组件，嵌入在**首页**或侧边栏中。使得首页内容也能每日小幅变动，进一步促进全站收录。

## 全新实战构想：基于 OKX 官方内容的“三层 SEO 漏斗矩阵”策略
*(注：计划优先在 `oyixiazai.com` 域名上进行落地试水)*

本策略旨在跳出单纯的“洗稿”思维，构建一个**健康、分层次、且高度自动化**的内容生态系统。通过三种相互联动的方案，实现引流、截流与转化的完美闭环。

### 层级一：动态资讯引流层（广撒网，养蜘蛛）
*   **定位**：网站的前锋，负责高频更新，保持网站活跃度（Freshness），吸引 Google 蜘蛛获取抓取配额（Crawl Budget）。
*   **执行**：监控 OKX 官方“最新公告”、“新币上线”等时效资讯。
*   **处理**：由 AI 执行极简的提炼总结（如 300字 左右的《快讯：XXX》），**不做长篇大论**，避免被判定为劣质内容。
*   **使命**：这些页面不需要排大词，唯一使命是在底部或侧边栏，通过密集的内链（Internal Links）将权重源源不断地导向“层级三”的核心单页。

### 层级二：长尾教程矩阵层（截流路人，稳基本盘）
*   **定位**：网站的中场，负责拦截大量的长尾搜索意图（如“怎么玩”、“手续费多少”、“防爆仓设置”等）。
*   **执行**：采集 OKX “新手学院”或帮助中心里的深度干货文章。
*   **处理**：让 AI 扮演“币圈老手”，将生硬枯燥的官方说明文，**结构化重构**为“大白话、实操向”的新手防坑指南。
*   **使命**：覆盖大量低竞争、高意图的长尾词。此类用户通常准备上手操作，是植入带渠道号的返佣链接/下载引导的绝佳场景。

### 层级三：核心流量变现层（精准打击，利润核心）
*   **定位**：网站的塔尖（核心 20% 页面），负责最终转化。
*   **执行**：瞄准“欧易下载”、“易欧App最新版”、“欧意注册官网”等民间俗称、高价值核心大词。
*   **处理**：**需要重度打磨**的终极权威单页（Pillar Pages）。不能仅靠流水线自动生成，需提取官方最新的 App 特性和奖励机制，以评测或终极指南的口吻呈现最高质量的内容。
*   **使命**：承接“层级一”和“层级二”输送的所有内链权重，集中火力攻克最难、最赚钱的搜索词，完成最终的下载/注册转化。

**结语**：这三层结构是一套完整的战术体系。运用自动化流水线搞定 80% 的引流与截流任务（层级一、二），将优质资源和人工算力聚焦在最核心的 20%（层级三），实现降维打击。
